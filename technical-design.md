# 技术方案文档

## 推荐技术栈

第一版使用纯静态前端：

- HTML：页面结构。
- CSS：视觉风格、响应式布局、动效。
- JavaScript：搜索、筛选、文章详情切换等交互。

原因：

- 个人博客第一版不需要后台和数据库。
- 静态网站部署简单，后续可以放到 Netlify、Vercel、GitHub Pages。
- 维护成本低，适合先把内容和视觉跑通。

## 项目目录结构

```text
.
├── index.html
├── requirements.md
├── technical-design.md
├── AGENTS.md
├── prompt.md
└── assets
    ├── css
    │   └── styles.css
    └── js
        └── app.js
```

## 前端实现方案

- 首页采用参考站式布局：
  - 顶部透明导航，滚动后变为白色毛玻璃导航。
  - 大幅首屏背景图，居中展示站点名称与副标题。
  - 主体为文章列表加右侧信息栏。
  - 卡片使用白底、圆角、柔和阴影和轻微悬浮动效。
- 文章数据先写在 `assets/js/app.js` 中，使用数组维护标题、分类、标签、日期、摘要和正文。
- 搜索、分类、标签筛选在浏览器内完成。
- 文章详情使用页面内详情面板或模态层展示，避免第一版做复杂路由。

## 后端实现方案

第一版不做后端。所有内容静态渲染在浏览器中。

后续如果文章数量变多，可以升级为：

- Astro 或 VitePress 生成静态博客。
- Markdown 文件作为内容源。
- 可选接入 Headless CMS。

## 数据库设计

第一版不使用数据库。

文章数据结构建议：

```js
{
  id: "js-event-loop",
  title: "JavaScript 事件循环学习笔记",
  category: "JavaScript",
  tags: ["JavaScript", "浏览器", "异步"],
  date: "2026-05-25",
  readTime: "8 min",
  excerpt: "文章摘要",
  content: "文章正文"
}
```

## API 设计

第一版不需要 API。

后续如果需要后台管理，再考虑文章列表、文章详情、分类、标签等接口。

## 登录 / 权限方案

第一版不需要登录。网站内容默认公开。

后续如果要做私密笔记，可以加：

- 管理端登录。
- 私有文章访问权限。
- 草稿状态。

## 支付方案

不需要支付功能。

## 文件上传 / 存储方案

第一版不做文件上传。图片使用公开图片链接或项目内静态资源。

后续如果写作中需要大量图片，可以使用：

- 项目静态资源目录。
- 对象存储。
- 图床服务。

## 部署方案

推荐部署到 Netlify 或 Vercel。

第一版部署方式：

- 直接上传静态文件。
- 或连接 Git 仓库自动部署。

## 环境变量

第一版没有环境变量。

## 第三方服务

第一版只使用公开 Web Font 与图标字体 CDN。若需要完全离线可再改成本地资源。

## 开发顺序

1. 创建静态页面骨架。
2. 实现参考站风格的首页视觉。
3. 添加文章数据和文章卡片渲染。
4. 添加搜索、分类、标签筛选。
5. 添加文章详情展示。
6. 做移动端适配。
7. 本地预览和修正细节。

## 容易踩坑的地方

- 过早做后台管理，会拖慢第一版上线。
- 过早引入数据库，会增加维护成本。
- 文章详情路由、评论、登录等功能不是 MVP 必需。
- 只模仿外观但没有内容结构，会导致网站像空模板。
- 移动端右侧栏需要自然下沉，不能挤压正文。

## 第一版不建议过度开发的地方

- 不做复杂主题系统。
- 不做多用户。
- 不做文章在线编辑器。
- 不做全站服务端搜索。
- 不做动画堆叠，保持轻量、清爽、可读。
