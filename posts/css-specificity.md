# CSS 选择器优先级详解

在前端开发和面试中，CSS 选择器及其优先级（Specificity，即“特异性”）是衡量一个候选人对浏览器渲染引擎理解程度的重要指标。

以下将通过**结构化分类**、**“四位坐标法”优先级计算模型**以及**大厂面试常考的边缘案例**，帮助你建立清晰的知识网络。

---

### 第一部分：CSS 选择器的系统分类

要准确计算优先级，首先需要对选择器进行精准分类。我们可以将常见的选择器分为以下几类：

1. **基础选择器**

    - **ID 选择器**：`#header`
    - **类选择器**：`.menu`
    - **属性选择器**：`[type="text"]`
    - **标签/元素选择器**：`div`, `p`
    - **通配符选择器**：`*`

2. **关系选择器**

    - **后代选择器**（空格）：`div p`（匹配 `div` 内的所有 `p`）
    - **子代选择器**（`>`）：`ul > li`（仅匹配 `ul` 的直接子级 `li`）
    - **相邻兄弟选择器**（`+`）：`h1 + p`（紧跟在 `h1` 后面的第一个 `p`）
    - **通用兄弟选择器**（`~`）：`h1 ~ p`（`h1` 后面所有的同级 `p`）

3. **伪类与伪元素（面试高频考点）**

    - **伪类（Pseudo-classes）**：表示元素的某种**状态**。使用单冒号 `:`。
    - 例如：`:hover`, `:focus`, `:first-child`, `:nth-child(n)`。
    - **伪元素（Pseudo-elements）**：表示虚拟的**具体位置**，能创建 DOM 树之外的抽象元素。标准推荐使用双冒号 `::`。
    - 例如：`::before`, `::after`, `::first-letter`。

### 第二部分：优先级计算模型（四位坐标法）

在浏览器中，样式的冲突是通过计算选择器的 **Specificity（特异性）** 来决定谁生效的。你可以将特异性想象为一个四位坐标系统：

`(A, B, C, D)`

从左到右，权重依次降低。只有当左边的数值相同时，才会比较右边的数值。

#### 1. 坐标位的赋值规则

| 坐标位 | 代表的含义 | 计数规则 | 示例 |
| --- | --- | --- | --- |
| **A** | **内联样式（Inline）** | 如果样式写在 HTML 的 `style` 属性中，计为 `1`，否则为 `0`。 | `<div style="color:red">` -> `(1,0,0,0)` |
| **B** | **ID 选择器数量** | 统计选择器中 `#` 的个数。 | `#nav #logo` -> `(0,2,0,0)` |
| **C** | **类、属性、伪类数量** | 统计 `.class`、`[attr]`、`:pseudo` 的个数。 | `.btn:hover[disabled]` -> `(0,0,3,0)` |
| **D** | **元素、伪元素数量** | 统计标签名 `div`、`::pseudo` 的个数。 | `div::before` -> `(0,0,0,2)` |

> **注：** 通配符 `*`、关系选择器（`+`, `>`, `~`, ` `）对优先级**没有任何贡献**，计为 `(0,0,0,0)`。

#### 2. 对比规则

比较两个选择器的优先级时，**从左向右逐位比较**。一旦在某一位上分出大小，后面的位数直接忽略。

- **案例 A：** `.list li` vs `#active`
    - `.list li` 有 1 个类、1 个元素 -> `(0, 0, 1, 1)`
    - `#active` 有 1 个 ID -> `(0, 1, 0, 0)`
    - 比较第二位（B 位）：`1 > 0`。因此 `#active` 胜出。

- **案例 B：** `(0, 0, 12, 0)` vs `(0, 1, 0, 0)`
    - 12 个类选择器连写（例如 `.c1.c2....c12`）能否超过 1 个 ID 选择器？
    - **不能。** 在规范中，B 位只要大于 0，C 位再高也无法逆袭（现代浏览器中各模块是隔离存储和比较的，不存在低位向高位进位的情况）。

#### 3. 特殊规则

- **`!important`**：它不属于上述四位坐标系统。它是“核武器”，会直接覆盖任何普通的选择器。只有当两个冲突的属性都有 `!important` 时，才回归到坐标系统比较。
- **继承（Inheritance）**：继承自父元素的样式，其优先级是**最低的**（甚至低于通配符 `*`）。任何直接选中该元素的样式都会覆盖继承的样式。

### 第三部分：面试官常考的“陷阱”与现代 CSS 特性

在面试中，大厂面试官往往不满足于简单的数值相加，他们会通过以下场景考查你的理解深度：

#### 1. 继承的陷阱

```HTML
<div id="parent" style="color: red;">
  <span class="child">文本内容</span>
</div>
```

```CSS
#parent { color: red; } /* 权重：(0,1,0,0) */
span { color: blue; }   /* 权重：(0,0,0,1) */
```

- **问：** 文本是什么颜色？
- **答：** 蓝色。虽然 `#parent` 的权重是 `(0,1,0,0)`，但 `span` 上的 `color` 属性是**继承**自父元素的。而 `span` 选择器是**直接选中**了该元素，哪怕权重只有 `(0,0,0,1)`，也大于继承的权重。

#### 2. 现代伪类 `:not()`, `:is()`, `:where()` 的优先级计算

这是近年来面试的常考热点：

- **`:not(selector)` 和 `:is(selector)`**：

它们本身不增加权重，但**括号内的选择器会参与权重计算**。

```CSS
div:not(.active) { ... }
/* 权重计算：div (0,0,0,1) + .active (0,0,1,0) = (0,0,1,1) */
```

对于 `:is()`，其权重等于**括号内参数中权重最高的那一个**。

```CSS
:is(.class, #id) { ... }
/* 无论实际匹配到哪个，整条规则的权重统一按参数中最高的 #id 计算，即 (0,1,0,0) */
```

- **`:where(selector)`**：

它的特异性**永远为 `0`** `(0,0,0,0)`，无论括号里写了什么。这在编写基础组件库时非常有用，方便外部使用者无痛覆盖样式。

```CSS
:where(#header) { color: red; }
/* 权重为 (0,0,0,0)，极易被外部的 .menu (0,0,1,0) 覆盖 */
```

#### 3. 样式表位置的影响（同权重合并）

如果两个选择器的权重**完全相同**，谁会生效？

- **规则：** 遵循“后浪推前浪”原则，在样式表中**后声明**的样式会覆盖先声明的样式。
- 注意：这与 HTML 中 class 属性里类名的先后顺序无关，只取决于 CSS 文件中样式规则出现的顺序。

### 面试回答模板

如果在面试中被问到“请讲讲 CSS 优先级”，你可以这样有条理地回答：

> “CSS 的优先级由**特异性（Specificity）**、**!important** 和**继承关系**共同决定。
> 首先，浏览器使用一个**四位坐标 `(Inline, ID, Class, Element)`** 来计算特异性。这四位从左到右依次比对，高位胜出，且不能进位。其中类、属性和伪类算在第三位，标签和伪元素算在第四位。
> 其次，有一些特殊情况需要注意：
> 1. **继承样式**的优先级低于任何直接指向元素的规则。
> 2. 现代 CSS 伪类中，`:not()` 和 `:is()` 的权重由括号内的选择器决定，而 `:where()` 的权重则恒为 0，这常用于提升组件库的样式可定制性。
> 3. 当特异性完全相同时，CSS 遵循**源码顺序中后声明者胜出**的原则。”
