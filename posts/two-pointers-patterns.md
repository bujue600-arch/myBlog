# 双指针（对撞、分离、快慢）

这份学习计划表对双指针算法的归纳非常系统。双指针的核心优势在于**空间复杂度的优化（通常为 `O(1)`）以及通过单次遍历（`O(N)`）减少不必要的重复计算**。

以下为您梳理双指针的三大题型、细分逻辑、算法本质、JS 模板，以及如何从高维视角去理解这一套双指针体系。

---

### 一、高屋建瓴：如何理解双指针的本质？

从高维视角来看，双指针的本质是**“降维”**与**“排除无效解空间”**。

1. **降维（Dimensionality Reduction）：**

在暴力解法中，我们常常需要两重循环（`O(N^2)` 复杂度）来遍历所有可能的位置组合 `(i, j)`。这在几何上相当于在一个二维矩阵中寻找答案。而双指针通过某些单调性或物理规律，让两个指针互不回溯地移动，将二维的搜索空间压缩到了 `O(N)` 的线性时间。

2. **排除无效状态（Pruning）：**

指针的每一次移动，都代表着我们**安全地过滤掉了一批不需要计算的无效组合**。例如在“盛最多水的容器”或“三数之和”中，移动指针意味着我们确信被跳过的那部分组合绝对不可能产生更好的解。

---

### 二、三大题型深析与 JS 模板

#### 1. 左右碰撞指针（Opposite Direction Pointers）

- **本质：** 两个指针从区间的**两端向中间**靠拢，通常用于处理**对称性、单调性（排序后）或最值边界**问题。

- **细分场景：**

    - **无条件/对称移动：** 指针同步向内收缩。如：验证回文串（125）、反转字符串（344）。
    - **有条件/贪心移动：** 比较两个指针指向的值，根据大小关系或特定规则，决定只移动左指针或只移动右指针。如：三数之和（15，利用单调性收缩）、盛最多水的容器（11，每次放弃较短的边界）、接雨水（42）。

- **JS 核心模板：**

```JavaScript
function collisionPointers(arr) {
    let left = 0;
    let right = arr.length - 1;

    while (left < right) {
        if (满足某种收缩条件 / 达到最优解) {
            // 处理逻辑
            left++;
            right--; // 或者只移动一侧
        } else if (当前状态偏大 / 需要调整) {
            right--; // 右指针左移
        } else {
            left++;  // 左指针右移
        }
    }
    return 结果;
}
```

---

#### 2. 分离指针（Two Pointers on Separate Sequences）

- **本质：** 两个指针分别在**两个不同的序列**（数组或字符串）上移动，用于解决**匹配、合并、子序列判定**等关联两组数据的问题。

- **细分场景：**

    - **从后往前遍历（逆向合并）：** 避免覆盖已有元素。如：合并两个有序数组（88）。
    - **同向匹配（同步/异步步进）：** 两个指针根据匹配情况决定谁移动。如：判断子序列（392，子序列匹配成功则子序列指针右移，主序列指针一直右移）。

- **JS 核心模板：**

```JavaScript
function separatePointers(arr1, arr2) {
    let p1 = 0;
    let p2 = 0;

    // 或者是 while (p1 < arr1.length || p2 < arr2.length) 视具体题目而定
    while (p1 < arr1.length && p2 < arr2.length) {
        if (arr1[p1] === arr2[p2]) {
            // 匹配成功的逻辑
            p1++;
            p2++;
        } else {
            // 不匹配时的逻辑，通常只移动其中一个
            p1++;
        }
    }
    return 结果;
}
```

---

#### 3. 同向快慢针（Same Direction / Fast-Slow Pointers）

- **本质：** 两个指针在**同一个序列上同向**移动，但**步长、触发移动的条件或起始时间不同**。常用于**原地修改数组**（减小空间复杂度）或**链表定位**。

- **细分场景：**

    - **读写分离（原地修改）：** 快指针（读指针）负责寻找符合条件的新元素，慢指针（写指针）负责记录下一个写入位置。如：移动零（283）、移除元素（27）、删除有序数组中的重复项（26）。
    - **步长/时间差（滑动窗口前身或链表定位）：** 快指针先走 `N` 步，然后快慢指针以相同速度前进，用于寻找特定相对位置。如：删除链表的倒数第 N 个节点（19）。

- **JS 核心模板：**

```JavaScript
function fastSlowPointers(arr) {
    let slow = 0; // 写指针/慢指针

    // 快指针（读指针）通常作为循环变量遍历整个数组
    for (let fast = 0; fast < arr.length; fast++) {
        if (满足保留/写入条件，例如 arr[fast] !== 0) {
            // 如果需要，这里可以做元素交换，或者直接覆盖
            // arr[slow] = arr[fast];
            [arr[slow], arr[fast]] = [arr[fast], arr[slow]];
            slow++; // 慢指针向前移动
        }
    }
    return slow; // 通常 slow 的位置代表新数组的长度
}
```

### 三、总结与做题心法

在实际做题时，您可以尝试通过以下思路来判断和应用双指针：

1. **观察数据结构与要求：**

    - 如果是**单数组且已排序**，或者求**两数之和/最值** -> 优先考虑**左右碰撞指针**。
    - 如果是**两个数组/字符串**交互 -> 优先考虑**分离指针**。
    - 如果是**原地修改数组**，要求 `O(1)` 空间，或者**链表求中点/环/倒数节点** -> 优先考虑**同向快慢指针**。

2. **明确指针的物理意义：** 在写代码前，先问自己：`left` / `slow` 代表什么？`right` / `fast` 代表什么？在循环过程中，这个物理意义必须保持**不变**（即维护循环不变量）。

3. **注意边界条件：** 指针移动是否会越界（`p1 < arr.length`），`left < right` 还是 `left <= right`，这些细节通常决定了程序是否能正确运行。
