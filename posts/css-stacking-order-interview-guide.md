# CSS 层叠顺序面试指南

在前端面试中，**层叠上下文（Stacking Context）**和**层叠顺序（Stacking Order）**是考察 CSS 功底的高频考点。面试官不仅看你是否知道 `z-index`，更看重你是否理解其背后的底层渲染机制。

为了便于理解和记忆，我们可以将这套复杂的 CSS 规则拆解为**一个核心比喻、两个关键概念、以及一套经典排序**。

---

### 一、核心比喻：“拼爹”理论（层叠上下文的层级关系）

在网页的 3D 视觉空间里，决定谁在上面、谁在下面，最核心的规律可以总结为：**“拼爹”理论**。

- **同一辈分比能力（`z-index` 值）**：如果两个元素处于同一个“层叠上下文”内，谁的 `z-index` 大，谁就在上面。
- **不同辈分拼父辈（层叠上下文本身）**：如果两个元素处于不同的“层叠上下文”中，那么它们自身的 `z-index` 再大也没有用。决定它们上下关系的是它们各自的**“父级层叠上下文”**的先后顺序。

> **面试经典场景：**
> 为什么我给元素 A 设置了 `z-index: 9999`，它还是被 `z-index: 1` 的元素 B 盖住了？
> **答：** 因为元素 A 所在的层叠上下文，其层叠等级比元素 B 所在的层叠上下文低。A 在自己的小圈子里是老大（9999），但放到大环境中，整个圈子都被 B 所在的圈子压在下面。

---

### 二、两个关键概念

#### 1. 层叠上下文（Stacking Context）

层叠上下文是 HTML 元素的三维自建物理空间。你可以把它想象成一个“独立的国家”。在这个国家内部，臣民们按照一定的法律（层叠顺序）排序，但一旦出国，外界只认这个国家元首的地位。

**如何触发（诞生）一个层叠上下文？**

常见的触发条件包括（面试时能列举出以下几类即可）：

1. **根元素**：`<html>` 天然是一个层叠上下文。
2. **定位元素与 z-index 结合**：`position` 值为 `absolute` 或 `relative`，且 `z-index` 值不为 `auto`（即具体数值）。
3. **现代定位属性**：`position: fixed` 或 `sticky`（在大部分现代浏览器中，它们天然创建层叠上下文，无需写 `z-index`）。
4. **CSS3 属性属性（常考点）**：
    - `opacity` 属性值小于 1。
    - `transform` 属性值不为 `none`。
    - `filter` 属性值不为 `none`。
    - `flex` 或 `grid` 容器的子元素，且子元素的 `z-index` 不为 `auto`。
    - `will-change` 指定了上述任意属性。

#### 2. 层叠水平（Stacking Level）

在同一个层叠上下文内，每个元素都有一个层叠水平。它决定了元素在 Z 轴上的先后顺序。普通元素和拥有层叠上下文的元素都可以比较层叠水平。

---

### 三、经典层叠顺序：七阶层叠水平（Stacking Order）

在一个确定的层叠上下文内部，各类型元素的从后往前（从屏幕深处向用户眼睛方向）的排列顺序如下。这就是著名的**“七阶层叠顺序表”**：

| 顺序（从后往前/底往顶） | 元素类型 | 解释与为什么这样设计 |
| --- | --- | --- |
| **1（最底层）** | 形成该上下文的元素的 **Background / Border** | 容器自身的背景和边框，自然应该在最底下。 |
| **2** | **`z-index` < 0** 的子元素 | 拥有定位属性且负 `z-index` 的元素。 |
| **3** | **块级盒子（Block）** | 无定位的块级元素（如 `<div>` 等），用于构建页面骨架。 |
| **4** | **浮动盒子（Float）** | 浮动元素（`float: left/right`）。 |
| **5** | **行内/行内块盒子（Inline/Inline-block）** | **关键点**：文字、图片等内容元素。CSS 的核心是展示内容，因此内容（Inline）默认要盖在骨架（Block/Float）之上，防止文字被背景或布局容器遮挡。 |
| **6** | **`z-index: 0` 或 `auto`** | 默认的定位元素。 |
| **7（最顶层）** | **`z-index` > 0** 的子元素 | 主动设置了正 `z-index` 的定位元素。 |

> **记忆小技巧：**
> 为什么 `inline` 元素比 `block` 和 `float` 还要靠前？
> 因为网页的核心是**内容（文字和图片）**，布局（Block）和装饰（Float）应当为内容让路，确保内容不被遮挡。

---

### 四、面试官评判标准与避坑指南（如何展示优异）

如果在面试中遇到相关问题，可以按照以下逻辑流畅回答，展现出系统性的思考：

1. **先亮出结论**：说明 `z-index` 不是决定元素层级的唯一因素，必须要先看它们是否处于同一个**层叠上下文**中。
2. **解释层叠上下文的创建**：主动提到除了 `position + z-index`，现代 CSS3 属性（如 `transform`、`opacity`、`flex` 等）也会创建层叠上下文。这能体现出你对现代 CSS 规范的掌握。
3. **背诵七阶顺序的核心逻辑**：不需要机械地背诵 1 到 7，而是向面试官阐述其背后的**设计哲学**（例如：“CSS 设计规范中，为了保证文本内容不被块级背景遮挡，行内元素 `inline` 的层叠水平高于块级元素 `block` 和浮动元素 `float`”）。
4. **给出解决方案**：当被问到如何解决“z-index 失效”的 Bug 时，说明排查步骤：
    - 第一步：检查该元素及其父元素是否创建了新的层叠上下文。
    - 第二步：利用浏览器的开发者工具（如 Chrome DevTools 的 3D View 或 Layers 面板）观察真实的层叠树。
    - 第三步：通过提升父级层叠上下文的 `z-index` 或将冲突元素移出当前的上下文来解决。
