# AGENTS.md

## 项目背景

这是一个个人博客兼知识库网站，用于公开展示作者的编程学习笔记和技术文章。目标用户包括作者本人、面试官、同行和朋友。

## 项目目标

第一版要做成一个可直接浏览的静态博客网站，排版和风格高度参考 `https://linzyblog.netlify.app/`：顶部导航、沉浸式首屏、文章卡片、右侧个人信息栏、分类、标签和归档信息。

## 技术栈约定

- 第一版使用 HTML、CSS、JavaScript。
- 不引入后端、数据库、登录系统。
- 不擅自替换为大型框架。
- 不引入不必要依赖。

## 目录结构约定

```text
.
├── index.html
├── requirements.md
├── technical-design.md
├── AGENTS.md
├── prompt.md
└── assets
    ├── css
    │   └── styles.css
    └── js
        └── app.js
```

## 代码风格约定

- HTML 结构语义清晰。
- CSS 使用变量维护主题色、阴影、圆角和间距。
- JavaScript 保持简单直接，优先使用小函数组织渲染和交互。
- 注释只写在不容易理解的地方。
- 文件内容默认使用 UTF-8。

## UI 风格约定

- 高度参考 `https://linzyblog.netlify.app/` 的布局气质。
- 明亮、清爽、卡片化。
- 首页有大幅背景首屏。
- 主内容区为文章卡片流，右侧为作者信息与站点统计。
- 使用圆角、柔和阴影、轻微悬浮动效。
- 移动端必须变为单栏，避免文字溢出。

## 开发流程约定

- 每次修改前先理解现有文件结构。
- 优先完成 MVP 主流程。
- 每一步修改后都要能在浏览器中看到效果。
- 不要一次性加入复杂功能。

## 文件修改规则

- 不要随意改动与当前任务无关的文件。
- 不要删除已有内容，除非用户明确要求。
- 如果文件已存在，先读取再判断是更新、补充还是覆盖。
- 不要擅自更换技术栈。

## 禁止行为

- 不要引入后台、数据库、登录、支付等第一版不需要的功能。
- 不要加入复杂构建流程。
- 不要把页面做成营销落地页，第一屏必须服务博客阅读体验。
- 不要让页面只有装饰没有内容。

## 测试与验证要求

- 本地打开或启动静态服务器验证页面。
- 检查桌面端和移动端布局。
- 验证搜索、分类筛选、标签筛选、文章详情打开和关闭。
- 检查浏览器控制台是否有明显错误。

## 每次改动后的说明格式

完成后说明：

- 改了什么。
- 为什么这样改。
- 如何验证。
- 是否还有后续建议。

## 不确定问题处理

产品问题可以询问用户，但一次只问一个非技术问题。

技术实现问题优先根据 `technical-design.md` 自行判断，不要把框架、数据库、部署、接口设计等技术选择抛给用户。
